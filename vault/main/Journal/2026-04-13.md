## 00:46 — Spurious test-routine warnings after deletion

- User asked why test-routine warnings kept appearing after the file was deleted
- Confirmed test-routine.md no longer exists; only reference is Obsidian cache (`.obsidian/workspace.json`), which is harmless and self-correcting
- Root cause: bot restarted at 00:20, 00:33, and 00:39 — each restart resets `_notified_invalid_routines` set, causing re-notification if the file was still cached or recently removed
- Last warning at 00:39:45 coincided with the 00:39 restart; no further warnings after that
- Concluded: spurious warnings were a side effect of multiple bot restarts, not a persisted error state
