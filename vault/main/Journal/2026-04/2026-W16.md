## What you'll find here
Semana de melhorias profundas no claude-bot (v3.37–v3.38.2) e nas pipelines de conteúdo. O bot ganhou cache de resposta pós-restart, restauração do modelo padrão do agente em `/clear`/`/new`, e correção do bug crítico `config is not defined` no update-check. Nova pipeline OSS Radar monitora repositórios GitHub diariamente com NO_REPLY automático. A rotina update-check foi reescrita (haiku→sonnet) para buscar e resumir changelogs reais do Claude Code CLI, repo e macOS app. No Jarvis Dashboard, adicionadas a grade Jellyfin e a aba Casa com cards de ACs via HomeBridge/SmartThings. O vault-health ganhou autocorreção automática de wikilinks. A técnica PinchTab `eval` foi descoberta como substituta definitiva de `text`+`snap` para extração estruturada de feeds do X e Threads.

## Goals
- Melhorar resiliência do bot em reinícios com cache de resposta
- Criar pipeline OSS Radar para monitoramento de repositórios GitHub
- Reescrever rotina update-check com fetch e resumo de changelog real
- Adicionar seção Jellyfin e aba HomeBridge/SmartThings ao Jarvis Dashboard
- Diagnosticar falhas de rate limit GLM em sessões paralelas
- Automatizar autocorreção de wikilinks no vault-health

## Decisions
- Response cache expira em 1h; restart recovery entrega texto cached em vez de "Interrompido"
- `/new` e `/clear` restauram modelo do frontmatter do agente, nunca herdam da sessão atual
- OSS Radar usa NO_REPLY quando nada relevante — sem notificação desnecessária
- update-check migrada haiku→sonnet (necessário para fetch + sumarizar changelog)
- PinchTab porta padrão é 9867 (`PINCHTAB_PORT`); `eval` + seletores CSS substitui `text`+`snap`
- HomeBridge cards usam `/api/accessories/layout` como fallback enquanto HAP cache vazio

## Progress
- Bot v3.37.0–v3.38.2: agent model restore, response cache, fix `config["claude_path"]` → `CLAUDE_PATH`
- Pipeline OSS Radar criada (collect haiku + analyze opus, NO_REPLY quando sem novidades)
- Rotina update-check reescrita: verifica CLI, repo, macOS app + resume changelog com botões inline Telegram
- Jarvis Dashboard: grid Jellyfin, badge X/Y eps, ícones legenda inline, HomeView.vue + aba Casa
- vault-health: tabela de mapeamentos e autocorreção de wikilinks antes de reportar
- PinchTab: porta 9870→9867 corrigida em todo o vault; técnica eval documentada em Tooling.md
