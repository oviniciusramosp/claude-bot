## 01:15 — Created operational skills library

- Discussed that routines/pipelines have been reinventing the wheel: each pipeline step embeds full logic for common operations (Notion API calls, image generation, web fetching). A bug in the Notion publishing logic would require editing every pipeline.
- Decision: introduce a layer of **operational skills** — atomic, reusable procedures that pipeline steps reference instead of re-implementing. Existing skills (create-routine, create-pipeline, etc.) are all about *creation*; operational skills cover *execution*.
- Created 6 new operational skills in Skills/:
  - publish-notion — Notion API, block conversion, 100-block batching, covers, retries
  - publish-x — PinchTab (preferred) + cookie-based GraphQL fallback, threading, media
  - publish-threads — PinchTab-only (no stable API), carousels, rate-limit defense
  - generate-image — Gemini nano-banana, prompt enhancement, catbox.moe hosting, dimension conventions
  - fetch-web — tool selection (PinchTab vs curl), RSS/HTML/JSON parsing, retries, rate limiting, caching
  - review-calendar — Finnhub economic calendar, SofaScore fixtures, BRT timezone handling, relevance scoring
- Each skill documents prerequisites from vault/.env, input contract, procedure, error handling, and references to existing pipeline implementations that match the pattern.
- Updated Skills/Skills.md index: split into "Creation skills" and "Operational skills" sections.
- Next step: migrate one existing pipeline step to reference these skills instead of containing inline logic — crypto-news/publish.md is the best starting point since it has the most duplicated Notion logic.
- No bot code changed — vault-only update, no version bump needed per CLAUDE.md policy.

## 01:40 — Migrated existing pipeline steps to reference operational skills

- Applied the new operational skills to 7 existing pipeline steps. Distinguished two migration strategies based on how much of each step was business logic vs infrastructure:
- **Full migration** (infrastructure replaced by skill reference, business params preserved):
  - crypto-news/steps/publish.md — Notion Python block replaced by reference to `publish-notion.md`; kept decision tree (relevant vs not), icon logic (📰/📡), two Telegram message templates, category/tags schema for this DB
  - crypto-ta-analise/steps/publisher.md — Notion Python block replaced by reference to `publish-notion.md`; kept "Análise DD Mês" title format, Category="Análise Técnica", Cover/Thumbnail URL properties, reviewer-notes comment posting
  - crypto-ta-analise/steps/cover.md — kept generate-ta-cover.py template script (correct per `generate-image.md` decision rule for template-based images with data overlays); added references to dimension conventions and catbox.moe pattern
- **Top reference only** (step is almost entirely business logic — DO NOT rip out):
  - palmeiras-feed/steps/publish.md — added reference to `publish-notion.md` but explicitly called out that the lock file, per-post dedup re-check, dual-key dedup (URL + normalized titulo), immediate state update, and 200-entry state cap are CRITICAL race-condition protections that must not be simplified
  - palmeiras-feed/steps/fetch-web.md — added reference to `fetch-web.md`; kept Gazeta/GE/Lance sources, keyword filters, 24h cutoff
  - palmeiras-feed/steps/fetch-images.md — added reference to `fetch-web.md` + note that `generate-image.md` is a future fallback but editorial reuse is preferred; kept full priority cascade (apiverdao, WP featured_media, WP search, recent media) and FACES player map
  - crypto-news/steps/collect.md — added reference to `fetch-web.md` and `review-calendar.md` for macro context; kept all Binance/alternative.me/crypto.news endpoints
- Key principle applied: **skills describe the "how"; steps describe the "what" with specific business parameters**. When a step's specifics encode past incident learnings (e.g., palmeiras-feed race conditions), those specifics stay — the skill reference is just a pointer to shared implementation context.
- No bot code changed — vault-only update, no version bump.
