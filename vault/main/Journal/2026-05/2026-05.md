## What you'll find here

Maio foi dominado pelo desenvolvimento acelerado do **MEXCBot**: o dashboard recebeu live entry/SL da MEXC, botão manual de BE com cache de trigger ao vivo, modal VIEW com chart Lightweight Charts e timeline de order events, paginação na tabela History, equity curves redesenhadas, e correção crítica do fan-out de sinais via **AutoTrader** (timeout granular + 2 retries). Em paralelo, o agente **crypto-bro** passou por refatoração profunda do pipeline `crypto-ta-analise-v2` — eliminando todos os `datetime.today()` distribuídos em favor de um `target_date.md` canônico e adicionando **Polymarket** como fonte de sentimento —, enquanto `crypto-ta-translate` ganhou freshness guard, consume-on-read e supressão de leaks Telegram. O **parmeirense** integrou GloboEsporte como fonte Tier 1 no feed e implementou do zero o pipeline `palmeiras-pos-jogo-v2` com detecção dual de fim de jogo (site oficial + SofaScore). No vault, a branch `agent-temp` foi criada para tornar arquivos temporários visíveis como nós intencionais no grafo Obsidian, e uma investigação revelou que restarts do bot eram causados pelo `backgroundtaskmanagementd` re-registrando launch agents — não crashes.

## Themes
- Evolução do MEXCBot como plataforma de trading completa (dashboard, UX, dados ao vivo)
- Robustez e anti-leaks em pipelines v2 (crypto-ta-analise, crypto-ta-translate)
- Novas fontes de dados (GloboEsporte, Polymarket, SofaScore)
- Vault structure: agent-temp branch, journal hierárquico, wikilinks path-qualified
- Automação pós-jogo para o parmeirense (palmeiras-pos-jogo-v2)

## Highlights
- MEXCBot modal VIEW: chart Lightweight Charts lazy-loaded com linhas Entry/SL/TP1/TP2, seletor de timeframe e timeline de order events
- `crypto-ta-analise-v2`: eliminação de `datetime.today()` distribuído via pre-step `compute-target-date` + `target_date.md` canônico; suporte a backdating por override
- `crypto-ta-translate` blindado: freshness guard 5min, consume-on-read de `args.md`, `output: none` em steps intermediários, banners de procedência no Telegram
- `palmeiras-pos-jogo-v2` implementada: 9 steps, detecção dual de fim de jogo, coleta de reações (Threads), artigos (GE/Verdazzo/NossoPalestra) e SofaScore (incidents/stats/escalações)
- AutoTrader fix: timeout granular (`connect=3s / read=30s`) + 2 retries com backoff — resolve falhas de fan-out silenciosas para MEXCBot
- GloboEsporte integrado no `palmeiras-feed` como Tier 1 com scraper customizado (sem RSS, extrai JSON embutido da listing page)
- Polymarket adicionado ao `crypto-ta-analise-v2` via Gamma API pública (8 markets, filtro volume ≥ $10k)
- MEXCBot live balance: `equity − unrealized` com cache 60s substitui snapshot diário defasado

## Decisions
- `target_date.md` como fonte única de verdade temporal no `crypto-ta-analise-v2`; nenhum script chama `date.today()` diretamente
- `args.md` one-shot: TTL 5min + delete-on-read previne reuso de URL stale na `crypto-ta-translate`
- `TRUSTED_SINGLE_SOURCES = {"palmeiras-api", "globoesporte"}` — bypass da regra de mínimo 2 fontes no palmeiras-feed
- `is_safe` calculado via cache `_SL_TRIGGER` ao vivo (TTL 30s) da MEXC, não valor stale do DB
- Startup notifications do MEXCBot delegadas inteiramente ao watchdog (transição down→up); bot não notifica em KeepAlive bounces
- Wikilinks path-qualified obrigatórios em parent links para evitar colisão de basename entre agentes no Obsidian

## Lessons
- `datetime.today()` distribuído em múltiplos scripts de pipeline cria dessincronização de título/capa/corpo quando o pipeline roda fora do horário canônico — centralizar em um pre-step com arquivo de estado é a solução correta
- `db.refresh(trade)` é obrigatório no início de funções SQLAlchemy chamadas após múltiplos `db.commit()` no mesmo ciclo — session expiry retorna objetos stale silenciosamente
- Pipeline v2 anti-leak: apenas `publish` steps com `sink` declarado devem chegar ao usuário; steps `llm`/`script`/`validate` que invocam `telegram_notify.py` bypassam as gates estruturais do harness

## Carry-forward
- `pos-jogo-scheduler`: rotina diária (06:00) que lê o calendário oficial e cria triggers precisos para dias de jogo, eliminando o schedule brute-force de 30min
- `collect_kalshi_v2.py`: skeleton espelhado no Polymarket para confluência de previsões de mercado no `crypto-ta-analise-v2`
- PinchTab health-check + auto-restart: daemon travou silenciosamente (stale Chrome process) — pipeline ou watchdog separado precisa detectar e reiniciar
- Sync reverso MEXCBot: atualizar `trade.sl_price` no DB periodicamente quando divergir do trigger ao vivo da MEXC, evitando acúmulo de stale entries
